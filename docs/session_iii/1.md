# Error Handling in Python

Error handling is the mechanism that allows your program to **gracefully recover from unexpected situations instead of crashing**.

Common scenarios:

* File not found
* Invalid user input
* Division by zero
* Network errors
* API failures
* Database connection failures

Coming from a .NET background:

| Python    | C#        |
| --------- | --------- |
| `try`     | `try`     |
| `except`  | `catch`   |
| `finally` | `finally` |
| `raise`   | `throw`   |

---

# 1. What Happens Without Error Handling?

```python
number = 10
result = number / 0

print(result)
```

Output:

```text
ZeroDivisionError: division by zero
```

The program stops immediately.

---

# 2. Using try/except

```python
try:

    number = 10

    result = number / 0

    print(result)

except ZeroDivisionError:

    print("Cannot divide by zero")

print("Program continues")
```

Output:

```text
Cannot divide by zero
Program continues
```

---

# 3. Catching Multiple Exceptions

```python
try:

    number = int(input("Enter a number: "))

    result = 100 / number

    print(result)

except ValueError:

    print("Please enter a valid number")

except ZeroDivisionError:

    print("Cannot divide by zero")
```

---

# 4. Generic Exception

```python
try:

    value = int("abc")

except Exception as ex:

    print(ex)
```

Output:

```text
invalid literal for int() with base 10: 'abc'
```

You can inspect:

```python
print(type(ex))
```

# 5. Exception Hierarchy

```text
Exception
│
├── ValueError
├── TypeError
├── KeyError
├── IndexError
├── FileNotFoundError
├── PermissionError
├── ZeroDivisionError
└── RuntimeError
```

Catch specific exceptions whenever possible.

Avoid this unless necessary:

```python
except Exception:
```

---

# 6. else Block

`else` executes only if no exception occurs.

```python
try:

    result = 100 / 5

except ZeroDivisionError:

    print("Error")

else:

    print("Success:", result)
```

Output:

```text
Success: 20.0
```

---

# 7. finally Block

`finally` always executes.

```python
try:

    print("Opening connection")

    result = 10 / 0

except ZeroDivisionError:

    print("Error occurred")

finally:

    print("Closing connection")
```

Output:

```text
Opening connection
Error occurred
Closing connection
```

Even if an exception occurs, `finally` runs.

# 9. Raising Exceptions (`raise`)

You can generate your own exceptions.

```python
salary = -100

if salary < 0:

    raise ValueError(
        "Salary cannot be negative"
    )
```

Output:

```text
ValueError: Salary cannot be negative
```

Equivalent C#:

```csharp
throw new ArgumentException();
```

---

# 10. Raising Exceptions in Functions

```python
def calculate_bonus(salary):

    if salary <= 0:

        raise ValueError(
            "Invalid salary"
        )

    return salary * 0.20
```

Usage:

```python
try:

    bonus = calculate_bonus(-1000)

except ValueError as ex:

    print(ex)
```

---

# 11. Creating Custom Exceptions

```python
class InvalidSalaryError(Exception):

    pass
```

Usage:

```python
def validate_salary(salary):

    if salary < 0:

        raise InvalidSalaryError(
            "Salary cannot be negative"
        )
```

```python
try:

    validate_salary(-100)

except InvalidSalaryError as ex:

    print(ex)
```

# 12. Resource Cleanup Example

Without cleanup:

```python
file = open("data.txt")

# exception occurs

file.close()
```

`close()` may never execute.

---

Better:

```python
try:

    file = open("data.txt")

    print(file.read())

finally:

    file.close()
```

---

Best:

```python
with open("data.txt") as file:

    print(file.read())
```

`with` automatically handles cleanup.

# 13. Practical Example

```python
def divide(a, b):

    try:

        result = a / b

        return result

    except ZeroDivisionError:

        print("Cannot divide by zero")

        return None

    finally:

        print("Operation completed")


result = divide(100, 5)

print(result)
```

Output:

```text
Operation completed
20.0
```

# 14. Realistic Employee Example

```python
def process_employee_salary(salary):

    if salary <= 0:

        raise ValueError(
            "Salary must be positive"
        )

    return salary * 0.20


def main():

    try:

        salary = int(
            input(
                "Enter salary: "
            )
        )

        bonus = process_employee_salary(
            salary
        )

        print(
            f"Bonus: {bonus}"
        )

    except ValueError as ex:

        print(
            f"Error: {ex}"
        )

    finally:

        print(
            "Program finished"
        )


if __name__ == "__main__":

    main()
```

### Example input

```text
120000
```

Output:

```text
Bonus: 24000.0
Program finished
```

# Recommended Error Handling Practices

## Catch specific exceptions

Good:

```python
except FileNotFoundError:
```

Avoid:

```python
except Exception:
```

unless absolutely necessary.

---

## Use `finally` for cleanup

```python
finally:

    connection.close()
```

---

## Use `with` for files

```python
with open(...) as file:
```

instead of manual close.

---

## Raise exceptions for invalid business rules

```python
if salary < 0:

    raise ValueError(
        "Invalid salary"
    )
```

---

## Don't silently ignore errors

Avoid:

```python
except:

    pass
```

because it hides problems.

# Error Handling in AI Applications

You'll frequently see:

```python
try:

    response = llm.invoke(prompt)

except TimeoutError:

    print("AI service timeout")

except Exception as ex:

    print(ex)
```

or

```python
try:

    data = json.load(file)

except json.JSONDecodeError:

    print("Invalid JSON")
```

These patterns are common when working with AI applications, APIs, and data pipelines built with frameworks such as FastAPI and AI libraries.
