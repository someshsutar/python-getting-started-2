# What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a programming style where you model your program as **objects**. Each object combines:

* **Attributes (data)** – what the object knows.
* **Methods (functions)** – what the object can do.

Think of a **Car**:

* Attributes: color, model, speed
* Methods: start(), stop(), accelerate()

Instead of writing separate functions that operate on data, OOP keeps the data and related functions together.

---

# Procedural Programming in Python

In procedural programming, data and functions are separate.

### Example: Bank Account

```python
# Data
balance = 1000

# Functions
def deposit(amount):
    global balance
    balance += amount

def withdraw(amount):
    global balance
    if balance >= amount:
        balance -= amount
    else:
        print("Insufficient balance")

deposit(500)
withdraw(200)

print(balance)
```

### Output

```
1300
```

### Problems

* Uses a global variable.
* Any function can modify `balance`.
* Hard to manage multiple bank accounts.

---

# Object-Oriented Programming in Python

Now let's create a `BankAccount` class.

```python
class BankAccount:

    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount

    def withdraw(self, amount):
        if self.balance >= amount:
            self.balance -= amount
        else:
            print("Insufficient balance")

    def show_balance(self):
        print(f"{self.owner}'s Balance = {self.balance}")


# Create objects
account1 = BankAccount("Alice", 1000)
account2 = BankAccount("Bob", 500)

account1.deposit(500)
account2.withdraw(200)

account1.show_balance()
account2.show_balance()
```

### Output

```
Alice's Balance = 1500
Bob's Balance = 300
```

Each account has its own data, and the methods operate on that specific account.

---

# Difference in Thinking

### Procedural

```
Data
 |
 +--> deposit()
 |
 +--> withdraw()
 |
 +--> show_balance()
```

Functions act on external data.

### Object-Oriented

```
BankAccount Object
------------------
owner
balance

deposit()
withdraw()
show_balance()
```

The object owns both its data and behavior.

# Another Example: Rectangle

## Procedural

```python
rectangle = {
    "length": 10,
    "width": 5
}

def area(rect):
    return rect["length"] * rect["width"]

print(area(rectangle))
```

Output:

```
50
```

---

## Object-Oriented

```python
class Rectangle:

    def __init__(self, length, width):
        self.length = length
        self.width = width

    def area(self):
        return self.length * self.width


rect = Rectangle(10, 5)

print(rect.area())
```

Output

```
50
```

The `area()` method naturally belongs to the `Rectangle` object.

# Why OOP is Better for Large Applications

Imagine building an online shopping system.

### Procedural

```python
customers = []
orders = []
products = []

def create_customer():
    ...

def create_order():
    ...

def create_product():
    ...
```

As the application grows, keeping track of shared data and related functions becomes more difficult.

---

### Object-Oriented

```python
class Customer:
    ...

class Product:
    ...

class Order:
    ...
```

Each class is responsible for its own data and behavior, making the system easier to understand and extend.

---

# Comparison

| Procedural Programming            | Object-Oriented Programming                         |
| --------------------------------- | --------------------------------------------------- |
| Focuses on functions              | Focuses on objects                                  |
| Data is separate from functions   | Data and methods are together                       |
| Easier for small scripts          | Better for large applications                       |
| Reusing code is more difficult    | Reuse through classes, inheritance, and composition |
| Changes can affect many functions | Changes are often isolated within a class           |

---

# When to Use Which?

**Use procedural programming when:**

* Writing small scripts
* Automating repetitive tasks
* Performing data processing
* Solving algorithmic problems

**Use OOP when:**

* Building web applications (such as with Flask or Django)
* Developing desktop or mobile applications
* Designing games
* Creating reusable libraries
* Managing complex systems with many interacting entities

---

## Python's Flexibility

One of Python's strengths is that you don't have to choose one paradigm exclusively. It's common to combine both styles:

* Use **classes** to model your application's core concepts (users, orders, products).
* Use **standalone functions** for utility tasks (formatting dates, validating email addresses, mathematical calculations).

This pragmatic approach is widely used in real-world Python projects and helps keep code both organized and easy to maintain.
