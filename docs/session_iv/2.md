Python fully supports the core concepts of Object-Oriented Programming (OOP). While Python doesn't enforce access restrictions like some other languages, it provides all the essential OOP features through its flexible and expressive syntax.

---

# 1. Class and Object

## Concept

A **class** is a blueprint for creating objects.

An **object** is an instance of a class.

### Real-world Example

Think of a **Car**.

A car blueprint specifies:

* Model
* Color
* Engine Capacity

Every manufactured car is an object created from that blueprint.

```
Class
-----
Car

Objects
-------
Car1
Car2
Car3
```

---

### Python Example

```python
class Car:

    def __init__(self, brand, color):
        self.brand = brand
        self.color = color

    def start(self):
        print(f"{self.brand} is starting")


car1 = Car("Toyota", "White")
car2 = Car("Honda", "Black")

print(car1.brand)
car2.start()
```

Output

```
Toyota
Honda is starting
```

# 2. Encapsulation

## Concept

Encapsulation means **combining data and methods into one unit (class)** and controlling access to the data.

Instead of changing data directly, users interact with it through methods.

Python uses naming conventions:

* Public → `balance`
* Protected (convention) → `_balance`
* Private (name mangling) → `__balance`

---

## Real-world Example

### ATM Machine

You cannot directly change your bank balance.

Instead, you use:

* Deposit
* Withdraw
* Transfer

The ATM protects the internal data.

---

### Python Example

```python
class BankAccount:

    def __init__(self, balance):
        self.__balance = balance

    def deposit(self, amount):
        self.__balance += amount

    def withdraw(self, amount):
        if amount <= self.__balance:
            self.__balance -= amount
        else:
            print("Insufficient funds")

    def get_balance(self):
        return self.__balance


account = BankAccount(1000)

account.deposit(500)
account.withdraw(200)

print(account.get_balance())
```

Output

```
1300
```

Trying this:

```python
print(account.__balance)
```

Produces

```
AttributeError
```

because `__balance` is private (name-mangled).

# 3. Abstraction

## Concept

Abstraction means **showing only essential features while hiding implementation details**.

The user knows **what** to do, not **how** it is done.

Python provides abstraction using the `abc` module.

---

## Real-world Example

### Driving a Car

You press:

* Accelerator
* Brake
* Clutch

You don't need to know how the engine, gearbox, and fuel injection work.

---

### Python Example

```python
from abc import ABC, abstractmethod

class Animal(ABC):

    @abstractmethod
    def sound(self):
        pass


class Dog(Animal):

    def sound(self):
        print("Bark")


class Cat(Animal):

    def sound(self):
        print("Meow")


dog = Dog()
dog.sound()
```

Output

```
Bark
```

Trying to create an `Animal` object directly:

```python
animal = Animal()
```

raises an error because `Animal` is abstract.

# 4. Inheritance

## Concept

Inheritance allows one class to reuse the properties and methods of another class.

It promotes code reuse.

---

## Real-world Example

```
Vehicle
   |
--------------------
|                  |
Car              Bike
```

Both Car and Bike inherit common vehicle properties like speed and fuel.

---

### Python Example

```python
class Vehicle:

    def start(self):
        print("Vehicle Started")


class Car(Vehicle):

    def open_sunroof(self):
        print("Sunroof Opened")


car = Car()

car.start()
car.open_sunroof()
```

Output

```
Vehicle Started
Sunroof Opened
```

---

### Types of Inheritance

Python supports all major types:

### Single

```
A
|
B
```

---

### Multiple

```
A     B
 \   /
   C
```

---

### Multilevel

```
A
|
B
|
C
```

---

### Hierarchical

```
      A
    / | \
   B  C  D
```

---

### Hybrid

Combination of multiple inheritance types.

# 5. Polymorphism

## Concept

Polymorphism means

**One interface, many implementations.**

The same method behaves differently for different objects.

---

## Real-world Example

### Payment Gateway

```
Pay()

Credit Card
UPI
Net Banking
Wallet
```

The customer always clicks **Pay**, but the underlying process depends on the payment method.

---

### Python Example

```python
class Dog:

    def sound(self):
        print("Bark")


class Cat:

    def sound(self):
        print("Meow")


animals = [Dog(), Cat()]

for animal in animals:
    animal.sound()
```

Output

```
Bark
Meow
```

The same method name (`sound`) produces different behavior depending on the object.

---

# 6. Method Overriding

## Concept

A child class provides its own implementation of a method inherited from its parent.

---

### Real-world Example

```
Employee

calculate_salary()

↓

Manager

calculate_salary()
```

Managers may receive additional bonuses, so they override the salary calculation.

---

### Python Example

```python
class Animal:

    def sound(self):
        print("Some sound")


class Dog(Animal):

    def sound(self):
        print("Bark")


dog = Dog()
dog.sound()
```

Output

```
Bark
```

The child class replaces the parent's implementation.

---

# 7. Method Overloading (Python Style)

Python does **not** support traditional method overloading (same method name with different parameter lists) as languages like Java or C# do. Defining multiple methods with the same name simply replaces the earlier definition.

Instead, Python typically uses **default arguments**, `*args`, or `**kwargs`.

### Example

```python
class Calculator:

    def add(self, *numbers):
        return sum(numbers)


calc = Calculator()

print(calc.add(2, 3))
print(calc.add(2, 3, 5))
print(calc.add(1, 2, 3, 4))
```

Output

```
5
10
10
```

The same method can handle a variable number of arguments.

# 8. Composition

## Concept

Composition models a **"has-a"** relationship, where one object contains another object. It is often preferred over inheritance because it creates looser coupling.

---

## Real-world Example

A **Car has an Engine**.

An engine is not a type of car, so inheritance would not make sense.

```
Car
 |
Engine
```

---

### Python Example

```python
class Engine:

    def start(self):
        print("Engine started")


class Car:

    def __init__(self):
        self.engine = Engine()

    def drive(self):
        self.engine.start()
        print("Car is moving")


car = Car()
car.drive()
```

Output

```
Engine started
Car is moving
```

# Summary of OOP Features in Python

| Feature                           | Purpose                                       | Real-world Example                        |
| --------------------------------- | --------------------------------------------- | ----------------------------------------- |
| Class                             | Blueprint for objects                         | Car blueprint                             |
| Object                            | Instance of a class                           | Your own car                              |
| Encapsulation                     | Protect data and expose controlled operations | ATM or Bank Account                       |
| Abstraction                       | Hide implementation details                   | Driving a car                             |
| Inheritance                       | Reuse existing functionality                  | Vehicle → Car, Bike                       |
| Polymorphism                      | Same interface, different behavior            | Different payment methods                 |
| Method Overriding                 | Customize inherited behavior                  | Manager salary calculation                |
| Method Overloading (Python style) | Handle different inputs with one method       | Calculator accepting any number of values |
| Composition                       | Build objects from other objects ("has-a")    | Car has an Engine                         |

### Key Takeaway

For beginners, remember these simple analogies:

* **Class** → Blueprint of a house.
* **Object** → An actual house built from the blueprint.
* **Encapsulation** → An ATM lets you perform transactions without exposing the bank's internal systems.
* **Abstraction** → You drive a car without understanding the engine internals.
* **Inheritance** → A `Car` is a specialized `Vehicle`.
* **Polymorphism** → The same **Pay** button works for UPI, cards, and wallets.
* **Composition** → A `Car` has an `Engine`; it is not an `Engine`.

These concepts form the foundation of writing clean, reusable, and maintainable object-oriented programs in Python.
