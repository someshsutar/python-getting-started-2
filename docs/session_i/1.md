# What are Modules in Python?

A **module** is simply a Python file (`.py`) that contains reusable code.

A module can contain:

* Functions
* Variables
* Classes
* Constants
* Executable code

Think of modules as a way to **organize code into separate files**.

Coming from a .NET background:

| Python            | C#                          |
| ----------------- | --------------------------- |
| Module (.py file) | Class file (.cs) or library |
| Package           | Namespace/Project           |
| Import            | using                       |

---

## Why Use Modules?

Imagine writing everything in one file.

**Without modules**

```text
main.py

5000 lines of code

- Database functions
- User functions
- Order functions
- Utility functions
- Email functions
```

Problems:

* Hard to maintain
* Difficult to debug
* Difficult to reuse
* Difficult to test

Modules solve this by splitting code.

```text
project/

main.py
user_service.py
order_service.py
email_service.py
utils.py
```

Benefits:

✅ Better organization

✅ Reusability

✅ Easier maintenance

✅ Easier testing

✅ Team collaboration

# Creating Your First Module

## calculator.py

```python
def add(a, b):
    return a + b


def subtract(a, b):
    return a - b


PI = 3.14159
```

---

## main.py

```python
import calculator

result = calculator.add(10, 20)

print(result)

print(calculator.PI)
```

Output:

```text
30
3.14159
```

---

# Import Specific Items

Instead of importing everything:

```python
import calculator
```

Import only what you need.

```python
from calculator import add

print(add(10, 20))
```

Output:

```text
30
```

---

# Import Multiple Items

```python
from calculator import add, subtract

print(add(10, 20))

print(subtract(20, 10))
```

---

# Alias Imports

```python
import calculator as calc

print(calc.add(10, 20))
```

Output:

```text
30
```

This is very common.

Example:

```python
import numpy as np
import pandas as pd
```

with NumPy and Pandas.

---

# Import Everything (Generally Avoid)

```python
from calculator import *
```

Then:

```python
print(add(10,20))
```

Works.

But avoid this because:

* Pollutes namespace
* Harder to debug
* Harder to understand where functions came from

---

# The `__name__` Pattern

### calculator.py

```python
def add(a,b):
    return a+b


if __name__ == "__main__":
    print(add(10,20))
```

### Running directly

```bash
python calculator.py
```

Output:

```text
30
```

### Importing

```python
import calculator
```

No output.

This prevents test code from running unintentionally.

# Creating a Real Project Structure

```text
crm_app/

main.py

services/
    user_service.py
    order_service.py

utils/
    validators.py
```

### user_service.py

```python
def get_user():

    return {
        "id":1,
        "name":"Someshwar"
    }
```

---

### main.py

```python
from services.user_service import get_user

def main():

    user = get_user()

    print(user)


if __name__ == "__main__":
    main()
```

# What is a Package?

A package is a folder containing modules.

```text
project/

services/
    user_service.py
    order_service.py
```

`services` is a package.

Modern Python often doesn't even require a special `__init__.py` file, although you'll still see it in many projects.

---

# How Python Finds Modules

Python searches in:

1. Current directory
2. Installed packages
3. System paths

You can see them:

```python
import sys

print(sys.path)
```
